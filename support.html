How to activate Wordpress theme?

This is 2 step process. First, you need to upload theme .zip file to Wordpress. Second, you need to activate uploaded theme.
After purchasing the theme from www.wptheme4.me you will get an email with URL to theme .zip file.

  • Download this .zip file to your computer.
  • Log in to your Wordpress Admin panel.
  • In 'Appearance' section select 'Themes' menu
  • Press 'Add new' button, press 'Upload Theme' button, press 'Choose file' button.
  • Select .zip file you downloaded to your computer and press 'Open' button.
  • To install theme press 'Install Now' button.
  • It will take up to 10s to install the theme. If everything goes as expected, you will get 'Theme installed successfully' message.
  • To activate uploaded theme press 'Activate' link. Message 'Theme is activated' indicates that your theme is activated.

Note: theme upload functionality is only available for self-hosted WordPress and Wordpress.com Business plan users. If youâ€™re using WordPress.com, then you will not see this option. See article 'WordPress.com and WordPress.org' that explains these differences.

If you want to switch from WordPress.com to self-hosted WordPress and need assistance see 'We provide services' section.

How to add or change a logo?

You have to be logged to Wordpress Admin panel.

  • In 'Appearance' section select 'Customize' menu
  • In 'Customizer' panel select 'Site Identity' section
  • Press 'Select logo' button.
  • In 'Select logo' modular window drop files from your computer OR press 'Select Files' button.
  • Upload selected file.
  • It is a good practice to add text in 'Alt text' field in 'Attachment details' section.
  • Press 'Select' button.
  • Press 'Skip Cropping' button if you do not need to crop logo image. Otherwise crop image and press 'Crop Image' button.
  • To save your changes permanently press 'Save & Publish' button in top left side of 'Customizer' panel

Note: logo image will have a link to your frontpage automatically

How to add images in frontpage?

Images in frontpage are post's 'Feature image'. Frontpage shows all posts from all categories in ascending order, i.e. newest post's 'Feature image' will be first in the row.

You have to be logged to Wordpress Admin panel.

  • In 'Posts' section select 'All posts' menu.
  • Find particular post in 'Posts' table and press 'Edit' link.
  • Select 'Set featured image' in 'Featured image' box.
  • Upload image file or choose one from 'Media library' and press 'Set featured image' button.
  • To save your changes permanently press 'Publish' button in 'Publish' box.

Tip: you want to change order of the images change post's publish date/time in Admin panel

How to change a number of images in frontpage?

You have to be logged to Wordpress Admin panel.

  • In 'Settings' section select 'Reading' menu
  • 'Reading settings' page has 'Blog pages show at most' field
  • To save your changes press 'Save Changes' button

Tip: for best visual layout on frontpage 'Blog pages show at most' field value should be equally divided from column number (2, 3 or 4 depending on the theme)

Tip: showing more than 20 images on the frontpage will increase site loading time for your visitors.

Note: altering 'Blog pages show at most' field will affect pagination in category pages as well

How to change post number on category pages?

You have to be logged to Wordpress Admin panel.

  • In 'Settings' section select 'Reading' menu
  • 'Reading settings' page has 'Blog pages show at most' field
  • To save your changes press 'Save Changes' button

Tip: for best visual layout on category pages 'Blog pages show at most' field value should be equally divided from column number (3 or 4 depending on the theme).

Tip: showing more than 20 images on each category page will increase page loading time for your visitors. It is recommended to show less images on each category page as pagination automatically will be added to navigate to the rest of category pages.

Note: altering 'Blog pages show at most' field will affect frontpage image number as well

How to make navigation menu?

Navigation menu is created automatically from the list of post categories that have posts attached to them (green line in picture below).

Categories that do not have any posts attached to them won't appear in navigation menu (reddish line in picture below).

You have to be logged to Wordpress Admin panel.

  • In 'Posts' section select 'Categories' menu
  • To create a new category fill 'Add New Category' form: 'Name' and 'Description' fields. Leave 'Slug' field empty as Wordpress will fill it automatically.
  • Press 'Add New Category' button to add it to the list of categories.

Attach post to a category

  • In 'Posts' section select 'All Posts' menu
  • In 'Edit Post' page attach post to category selecting category name in 'Categories' section in the right side of the page.
  • Press 'Update' button.

Visit your site and you will see navigation menu updated with your category name

Note: navigation menu has ascending order

Are your themes secure?

WPScan Vulnerability database reports that Wordpress plugins are the biggest source of vulnerabilities in Wordpress (54% of the global WordPress vulnerabilities count). Themes vulnerabilities and Wordpress core vulnerabilities counts 15% and 31% respectively.

Developing Wordpress themes we focus only for essential functionality. This approach provides a lot of advantages in terms of performance and security. Adding just enought functionality we reduce threat surface.

WPTheme4.me themes are developed with respect to Wordpress coding standards, best practises, security checks and do not use any 3rd party plugins.

We guarantee that WPTheme4.me themes:

  • .zip file viruse free
  • do not have malicious or potentially unwanted code.
  • do not use deprecated functions
  • HTML5 and CSS3 valid

Recommendations to secure your Wordpress site:

  • Keep Wordpress core up to date
  • Use as little 3rd party plugins as possible
  • Check plugins for vulnerabilities in WPScan Vulnerability database before activating in production
  • Check themes with Theme Authenticity Checker before activating in production
